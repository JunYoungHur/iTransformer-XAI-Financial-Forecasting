# iTransformer-XAI-Financial-Forecasting

미국 주식의 방향성을 변수 축 attention(iTransformer 계열)으로 예측하고,
모델이 무엇을 보고 판단했는지(XAI)를 분석하는 개인 프로젝트.

학부 팀 프로젝트 [tft-stock-forecasting](https://github.com/JunYoungHur/tft-stock-forecasting)의
후속 연구다. 그 프로젝트가 남긴 두 숙제 — 이종 변수가 한 토큰에 뭉개지는 구조,
그리고 예측 근거를 설명할 수 없다는 점 — 에 답해보려는 시도다.

> 완성된 예측 시스템이 아니라 진행 중인 연구다. 도달한 지점과 부딪힌 문제,
> 다음 방향을 함께 기록한다.

---

## 배경

24년 팀 프로젝트는 TFT로 종가를 예측하는 회귀 모델이었다. 진행하면서 두 가지가
계속 걸렸다.

- **변수 뭉개짐**: TFT는 한 시점의 모든 변수를 하나의 토큰으로 합쳐 시간 축에
  attention한다. 금리·거래량·종가처럼 성격이 다른 변수가 한 토큰 안에서 섞인다.
- **설명 불가**: 예측이 맞아도 왜 그렇게 판단했는지 알 수 없었다.

이후 [iTransformer](https://arxiv.org/abs/2310.06625)가 첫 번째 문제를 정면으로
다룬다는 것을 알게 됐다. 시간 축이 아니라 변수 축에 attention을 거는 접근이다.
이 프로젝트는 그 위에서 두 번째 숙제인 설명 가능성까지 함께 다룬다.

## 설계

**회귀 대신 분류로.** 종가 회귀 대신, 변동성 기반 임계값으로 라벨링한 3진 방향 분류
(하락/횡보/상승, 21영업일)를 택했다. 이유는 세 가지다. (1) 낮은 SNR에서 MSE는
출력이 직전 값으로 수렴하는 lagging이 강했다(직접 관측). (2) 원시 가격은
비정상적이지만 방향은 가격 수준에 불변이다. (3) long/short/hold 판단에는 크기보다
방향이 중요하다.

**변수 축 attention.** 각 변수의 최근 40일치 시계열을 토큰 하나로 임베딩하고, 변수
토큰 사이에 attention을 건다. "변수 뭉개짐" 숙제에 대한 답이다.

**학습된 변수 선택 게이트.** 어떤 변수가 중요한지를 사람이 직접 고르면 주관이
개입한다. 그래서 사람의 사전 선택을 최소화하고, 무엇을 중요하게 볼지의 판단을
모델에 맡겼다. TFT의 Variable Selection Network(VSN)를 변수 토큰화 직전에 두어
노이즈 변수를 누른다. 여기에 GRU를 결합해, 변수 중요도가 고정되지 않고 시점까지의
시장 맥락에 따라 달라지도록 했다. 모델이 어떤 변수에 무게를 두는지 들여다보려는,
"설명 가능성" 숙제에 대한 시도다.

**정규화 (RevIN).** 주가는 비정상적(non-stationary)이라 시기마다 평균·분산이
다르다. 전역 통계로 정규화하면 test 구간 통계가 섞이는 leakage가 생기고, train
통계로 고정하면 분포 변화에 취약하다. [RevIN](https://openreview.net/forum?id=cGDAkQo1C0p)은
각 입력 윈도우(인스턴스)를 그 자체 통계로 정규화해 두 문제를 동시에 피한다.
윈도우 내부 통계만 쓰므로 leakage가 없고, 시기별 분포 변화에 자동으로 적응한다.

**파이프라인**

```
RevIN  →  GRU 기반 변수 선택 게이트  →  변수-as-토큰 임베딩
       →  Transformer encoder (변수 축 attention)  →  GAP  →  MLP 분류기
```

**데이터.** Yahoo Finance S&P 500 일봉(2000~현재), 원본 가격(OHLV) 포함 47개 피처.

- 종목별 기술 지표: RSI, MACD, ATR, Bollinger, ROC, Stochastic, Williams %R,
  ADX, CCI, OBV, MFI, CMF, 거래량 이상치
- 거시: VIX, 금리, 달러, 금, 원유, 장단기 금리차
- 섹터 상대강도: 8개 SPDR ETF의 SPY 대비

관련 있을 법한 47개를 사전 선택 없이 모두 투입하고, 무관한 변수를 누르는 일은
학습된 게이트에 맡겼다.

## 결과

Hold-out 테스트셋, horizon=21, Colab T4 환경. 랜덤 베이스라인 33.3%.
아래 표의 "가중치"는 클래스 불균형을 보정하는 손실 가중치를 뜻한다.

| 설정 | Test Acc | 상위 10% 확신 베팅 정확도 | 베팅 long 비율 |
|---|---:|---:|---:|
| GRU 게이트 · 가중치 없음 | 38.2 % | 38.2 % | 49.6 % |
| GRU 게이트 · inverse 가중치 | 39.4 % | 13.8 % | 37.0 % |
| 시점 독립(pointwise) 게이트 · 가중치 없음 | 33.8 % | 23.3 % | 1.2 % |

정확도는 랜덤을 크게 넘지 못했다. 21일 뒤 방향을 맞히는 일은 그만큼 어려웠고,
이 프로젝트에서 정확도 자체를 성과로 내세우긴 어렵다. 대신 그 과정에서 관찰한
세 가지를 기록한다.

- **클래스 가중치의 역설.** 클래스 불균형을 보정하려고 빈도가 적은 클래스에 더 큰
  손실 가중치를 주는 inverse 방식을 썼는데, 오히려 예측 분포가 뒤집혀 하락을
  데이터 비율(~26%)보다 많은 ~55%로 예측했다. 가중치를 빼니 예측이 균형을 찾고
  확신도도 의미를 가졌다. 정확한 원인까지 규명하지는 못했지만, 신호가 약한
  데이터에서는 이런 보정이 오히려 역효과를 낼 수 있다는 사례로 남겨 둔다.
- **게이트 종류 비교.** 같은 조건(가중치 없음)에서 GRU 게이트가 시점 독립
  게이트보다 정확도가 높고(38.2% vs 33.8%), 확신 베팅의 long/short 균형도 훨씬
  낫다(49.6% vs 1.2% — 시점 독립 게이트는 확신 베팅이 거의 한 방향으로 쏠렸다).
  단일 실행 기준이라 경향성 수준의 관찰이며, seed 반복 검증은 남은 과제다.
- **게이트가 보는 변수.** 학습된 게이트는 변동성(ATR), 거시 국면(VIX, 장단기
  금리차), 방어 섹터에 무게를 둔다. 경제적으로 납득되는 묶음이다. 다만 이는 게이팅
  단계의 강조도일 뿐, 이후의 변수 축 attention이 이 가중치를 다시 다루므로 최종
  예측의 근거와 그대로 일치한다고 보기는 어렵다.

아래는 예시 종목 하나(티커 A)에 대한 시점별 변수 가중치(VSN) 변화다. 왼쪽(GRU)은

## 한계

- **생존 편향.** 종목 리스트가 *현재* S&P 500 구성이다. 2000년 이후 상장폐지·편출
  종목이 빠져 결과가 낙관적이다. 보정하려면 시점별 구성종목 데이터가 필요한데,
  상폐 종목은 무료 API에서 데이터가 끊기거나 제공되지 않아 현재 범위에선 보정이
  어렵다.
- **짧은 out-of-sample.** 학습에 쓰지 않은 진짜 검증 구간이 몇 개월뿐이다. 단일
  구간의 결과는 그것이 일반적인지 우연인지 가리기엔 약한 증거다.
- **낮은 SNR.** 21일 방향은 노이즈에 가깝다. 정확도가 33~39%에 머문 것도 이
  때문으로 보인다.
- **불완전한 확신 보정.** 최선의 실행에서도 가장 확신이 높은 구간에서 정확도가
  떨어졌다.
- **해석의 범위.** 게이트 가중치는 강조도이지 인과적 기여도가 아니다.

## 의의

성능 자체보다, 그 주변에서 시도한 구조적·방법론적 고민에 의미를 둔다.

- 이종 금융 변수를 한 토큰에 섞지 않고 변수-as-토큰으로 다뤘다.
- 맥락을 반영하는 GRU 게이트가 정적 게이트와 실제로 다르게 동작하는지 검증하고,
  숫자만 보고하는 대신 그 동작을 직접 들여다봤다.
- 앞선 팀 프로젝트의 두 숙제(변수 뭉개짐, 설명 가능성)에 구체적인 답을 시도하고
  그 한계까지 정리했다.

## 발견된 문제와 개선 방향

- 확신이 높은 구간에서 정확도가 더 보장되지 않는 문제 → 확신도와 실제 정확도를 맞추는 보정(calibration) 검토
- 단일 테스트 구간의 약한 증거 → 여러 구간으로 나눠 굴리는 rolling 평가
- 게이트 종류 비교가 단일 실행 기준 → 여러 seed 반복으로 경향의 통계적 확인
